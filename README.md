# Chirpy API 🚀

Hi 👋  
This project is a backend REST API called **Chirpy**, built as part of my learning journey in backend development.

I developed this project to practice real-world backend concepts such as authentication, database handling, API design, and debugging production-like issues.

---

## 📌 What This Project Does

Chirpy is a simple social-style API where users can:

- Create an account
- Log in securely using JWT authentication
- Post short messages called "chirps"
- View chirps (with sorting support)
- Delete their own chirps
- Refresh authentication tokens
- Handle webhook events (for upgrading users)

---

## 🧠 Why This Project Matters

Although this project was guided, I focused on deeply understanding and implementing the logic myself.  

During development, I faced and solved real backend problems such as:
- Handling authentication securely (JWT + refresh tokens)
- Fixing database state issues (reset bugs, duplicate data)
- Implementing sorting logic correctly
- Debugging API errors (500 responses, incorrect outputs)

This helped me move beyond just coding — into thinking like a backend engineer.
